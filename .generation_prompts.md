# Historial de Prompts (Generación de VitalTrack MVP)

A continuación se enumeran cronológicamente los prompts utilizados durante el desarrollo del proyecto para asegurar la trazabilidad del proceso:

1.  **Alineación Inicial**: Leer archivos `context.md` y `scope.md`, confirmar stack tecnológico y resumir pilares de arquitectura.
2.  **Configuración de Proyecto**: Actualizar `pubspec.yaml` con dependencias (Drift, Freezed, etc.) y crear estructura de directorios Clean Architecture.
3.  **Capa de Autenticación**: Implementar dominio y datos de Auth (Firebase) con navegación reactiva y tests unitarios (`mocktail`).
4.  **Modernización de Dependencias**: Ejecutar `flutter pub upgrade` y restaurar `flutter_gen_runner` resolviendo conflictos de Riverpod 3.x.
5.  **Capa Core (Routing & Theme)**: Configurar `GoRouter` con `refreshListenable` y tematización Material 3 (Verde #4caf50) para Cards y FormFields.
6.  **Persistencia Local**: Implementar tablas Drift, `HealthLocalDataSource`, repositorios y `AsyncNotifier` para el Dashboard con tests.
7.  **Pantalla de Login**: Desarrollar `LoginScreen` con validaciones, i18n, assets y manejo de errores mediante SnackBar.
8.  **Pantalla de Registro**: Implementar `RegisterScreen` con reglas de seguridad de contraseña estrictas y validación de coincidencia.
9.  **Dashboard (Bento Box)**: Crear interfaz de métricas responsiva con FAB expandible y manejo de estados de carga/vacío.
10. **Formulario de Entrada**: Implementar `AddEntryForm` con validaciones de co-dependencia para ejercicios y persistencia en Drift.
11. **Historial de Métricas**: Desarrollar `HistoryScreen` filtrada por tipo, con visualización detallada y navegación interactiva.
12. **Refactorización de Metadatos**: Crear clase `sealed HealthMetricInfo` para centralizar iconos, unidades y colores en toda la app.
13. **Refinamiento Bento**: Ajustar layout de la celda de Ejercicio (ancho completo y posición del tipo de actividad).
14. **Salud Nativa (Health API)**: Implementar integración con HealthKit/Google Fit, segmentación por horas y sincronización en segundo plano.
15. **Arquitectura Background**: Refactorizar lógica de `Workmanager` hacia la capa de datos con abstracciones de dominio.
16. **Optimización de Startup**: Mejorar `main.dart` con manejo de errores fatales y flujo de inicialización resiliente.
17. **Borrado Granular**: Implementar funcionalidad "deslizar para borrar" que limpie propiedades específicas sin eliminar el registro completo.
18. **Ajustes de UI (Dismissible)**: Refactorizar `_HistoryItem` eliminando el `Card` para asegurar alineación perfecta en la animación de borrado.
19. **Exportación JSON**: Crear caso de uso para exportación masiva del historial en formato `snake_case` filtrado por fecha.
20. **Gestión de Errores Nativa**: Implementar diálogo de error de permisos nativos y configuración de `Info.plist`/`AndroidManifest`.
21. **Compartir Historial**: Añadir acción al FAB para compartir el JSON de los últimos 7 días mediante `share_plus`.
22. **Compatibilidad Multiplataforma**: Verificar ejecución en iOS, Android y Web, documentando bloqueadores técnicos.
23. **Localización de Errores**: Internalizar todos los mensajes de excepción de Firebase Auth para soporte multi-idioma.
24. **Limpieza Final**: Auditoría de dependencias, eliminación de paquetes no usados y activos redundantes.
25. **Documentación**: Actualizar `README.md` con retrospectiva técnica, uso de IA y guía de ejecución.
26. **Retrospectiva**: Crear documentación de mejoras en `.improves/` y consolidar el resumen de aprendizaje del MVP.
